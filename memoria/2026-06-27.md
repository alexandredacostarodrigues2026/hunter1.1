# Atualização 2026-06-27

## Resumo Executivo

Reconstrução do "Equalizador de Produtos" como app portátil (Streamlit) dentro de `OPERACOES/geraldo_2020_2024/ESSENCIAL/`: scaffold inicial, ingestão direta de NF-e (XML) e SPED/EFD sem depender de exportação Qlik, identificação automática da entidade auditada por CNPJ, painel de upload/carga de XML classificado em ET/EP, fluxo multi-operação via `.bat`, e correção crítica de portabilidade do runtime Python (trocado de venv-Anaconda para distribuição embeddable real). Histórico completo e detalhado de cada etapa (PS0–PS9) está em `OPERACOES/geraldo_2020_2024/ESSENCIAL/history/alteracoes/CHANGELOG.md` — este registro é um resumo para a pasta `memoria/` recém-adotada.

## Funcionalidades Implementadas

- Scaffold portátil `ESSENCIAL/` (app/, banco/, config/, runtime/, launcher/, history/, logs/, qlik/, etc.) seguindo `PROJETO20260613.txt` (PS0).
- `loader.py`: ingestão direta de NF-e (`1-DOCFISCAIS/nf/*.txt`) e SPED/EFD (`2-DECLARACAO/SPED/*.txt`), substituindo a leitura via CSV do Qlik (PS1→PS2).
- Identificação automática da entidade auditada (CNPJ/Razão Social) por frequência entre pastas `ET/` (destinatária) e `EP/` (emitente), fixada em `config.json` (PS3).
- Painel Streamlit (`main.py`) exibindo a entidade auditada (PS4).
- Upload de XML de NF-e via arraste-e-solte, classificado automaticamente em ET/EP conforme o CNPJ fixado (PS5) — depois substituído por carga automática ao apontar a operação (PS6).
- Estrutura multi-operação: `OPERACOES/<operacao>/` com motor Python compartilhado; script `carregar_xml.bat` por operação (PS7→PS8); pasta-modelo `OPERACOES/_MODELO_OPERACAO/` para onboarding de novas operações.

## Correções Realizadas

- `_localizar_arquivos_nfe` usava `glob` não-recursivo e parou de achar os arquivos após reorganização em `ET/`/`EP/` — corrigido para `rglob` (PS3).
- `salvar_entidade_auditada()` não persistia `total_linhas_analisadas`/`por_fonte`, causando exibição incorreta no painel — corrigido (PS4).
- Bug de índice no parsing do registro `0000` do SPED (campo `DT_INI`) deixava `COMPETENCIA` vazio — corrigido (PS2).

## Alterações de Arquitetura

- Abandono definitivo da exportação via Qlik como fonte de dados (decisão do usuário) — ingestão 100% direta de NF-e/SPED brutos.
- Reorganização do projeto: `geraldo_2020_2024/` movida para `OPERACOES/geraldo_2020_2024/`, alinhando com o padrão `OPERAÇÕES/<operacao>/` do `PROJETO20260613.txt`.
- `loader.py` ganhou suporte a `HUNTER_OPERACAO_DIR` (variável de ambiente) para apontar o mesmo motor Python a qualquer pasta de operação, sem duplicar código.

## Alterações de Infraestrutura — CRÍTICO

- **`ESSENCIAL/runtime/` não era portátil de verdade**: era um `venv` criado a partir do Anaconda local da máquina de desenvolvimento (`pyvenv.cfg` apontava para `C:\Users\...\anaconda3`). Reconstruído com a distribuição **Python 3.12.3 embeddable oficial** (python.org) + dependências instaladas dentro dela — agora funciona em qualquer máquina Windows sem Python/Anaconda instalado (PS9).
- Corrigido vazamento para pacotes do perfil do usuário do Windows (`%APPDATA%\Python\...`) via `PYTHONNOUSERSITE=1`, aplicado em todas as invocações do runtime.
- Removidos caracteres não-ASCII de todos os `.bat` (causavam corrupção de parsing em certos contextos de execução).
- `runtime/` antigo preservado em `runtime_OLD_venv_backup/` (rollback).

## Problemas Encontrados

- `.bat` invocados de forma não-interativa (ferramentas de automação) corrompiam por causa de `chcp 65001` + caracteres acentuados — investigado e confirmado como risco real (não só artefato de teste); corrigido removendo acentuação de todos os `.bat`.
- Processos Streamlit órfãos (o `launcher_exe.py` desacopla o subprocesso de propósito) acumularam-se durante testes manuais, causando resultados inconsistentes — mitigado encerrando processos antes de cada novo teste.
